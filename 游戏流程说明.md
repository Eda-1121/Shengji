# 升级（拖拉机）游戏流程说明

## 游戏流程概览

```
初始化游戏 → 开始新一局 → 循环游戏 → 游戏结束
```

---

## 详细流程

### 1️⃣ 初始化游戏 (initialize_game)

**操作**:
- 创建牌堆（2副或4副牌）
- 创建4名玩家（1人类玩家 + 3 AI玩家）
- 分配队伍：玩家1和3一队，玩家2和4一队
- 初始化UI

**进入**: 步骤2

---

### 2️⃣ 开始新一局 (start_new_round)

#### 步骤1: 重置游戏状态
- 清空分数、手牌
- 重置叫牌状态
- 设置游戏阶段为 `DEALING_AND_BIDDING`

#### 步骤2: 洗牌
- 创建牌堆（2副牌 = 108张，4副牌 = 216张）
- 打乱牌序

#### 步骤3: 准备底牌
- 从牌堆顶部抽取**8张牌**作为底牌
- 剩余牌用于发牌

#### 步骤4: 开始发牌（发牌过程中可叫牌）
- 从庄家开始顺时针逐张发牌
- 每发一张牌，检查玩家是否拿到等级牌
- 如果拿到等级牌，可以随时叫牌

**叫牌规则**:
- **首次叫牌**: 只需1张等级牌
- **反主/加固**: 需要比当前叫牌更多的等级牌
  - 同队加固: 相同花色 + 更多张数
  - 对手反主: 任意花色 + 更多张数

#### 步骤5: 发牌完成后的最后叫牌机会
- 所有牌发完后，再给一次叫牌机会
- 检查所有玩家是否有更多等级牌可以反叫

#### 步骤6: 确定主牌和庄家
- **叫牌成功**: 最后叫牌的玩家成为新庄家，其叫的花色为主牌
- **无人叫牌**: 默认庄家队叫黑桃

**进入**: 步骤7

---

### 3️⃣ 埋底阶段 (start_burying_phase)

#### 步骤7: 庄家埋底
- 庄家收到8张底牌（手牌总数 = 发牌数/4 + 8）
- 庄家从手牌中选择8张牌扣底
- **人类玩家**: 手动选择8张
- **AI玩家**: 自动选择最小的8张

**埋底完成**:
- 庄家手牌数 = 发牌数/4（与其他玩家相同）
- 底牌保存，等待最后结算

**进入**: 步骤8

---

### 4️⃣ 出牌阶段 (start_playing_phase)

#### 步骤8: 初始化出牌阶段
- 设置游戏阶段为 `PLAYING`
- 更新所有牌的主牌状态
- 重新排序手牌（主牌放最后）

#### 第1轮出牌
**首家**: 庄家（叫牌成功的玩家）先出牌

**出牌顺序**: 顺时针（庄家 → 下家 → 对家 → 上家）

#### 后续轮次出牌
**首家**: 上一轮的赢家先出牌

**出牌规则**:
1. **首家出牌**: 可以出任意合法牌型（单张、对子、拖拉机、甩牌）
2. **跟牌**: 必须遵循跟牌规则（详见游戏规则文档）

#### 本轮结算 (evaluate_trick)
- 4名玩家都出牌后，判定本轮赢家
- 赢家获得本轮所有分数（5、10、K）
- 清理本轮出的牌

**循环**:
- 如果手牌还有剩余 → 继续下一轮（赢家先出牌）
- 如果所有手牌出完 → 进入步骤9

---

### 5️⃣ 结算与升级 (end_round)

#### 步骤9: 底牌结算
- 最后一轮的赢家队伍获得底牌分数
- **底牌倍数**: 2倍（如果用对子或拖拉机赢最后一轮可以更高）

**底牌归属**:
- 庄家队赢最后一轮 → 庄家队扣底（+底牌分×2）
- 对手队赢最后一轮 → 对手队抠底（+底牌分×2）

#### 步骤10: 升级计算

**升级规则**（根据对手队得分）:

| 对手得分 | 结果 | 庄家 |
|---------|------|------|
| < 40分 | 庄家升3级 | 不变 |
| 40-79分 | 庄家升2级 | 不变 |
| 80-119分 | 庄家升1级 | 不变 |
| 120-159分 | 对手升1级 | **轮换** |
| 160-199分 | 对手升2级 | **轮换** |
| ≥ 200分 | 对手升3级 | **轮换** |

**注意**: 对手得分≥120时，庄家会轮换到对手队

#### 步骤11: 检查游戏是否结束
- **游戏结束**: 任意队伍等级达到A（14级）
- **继续游戏**: 返回步骤2，开始新一局

## 牌型说明

| 牌型 | 说明 | 示例 |
|------|------|------|
| **单张** | 1张牌 | 5♠ |
| **对子** | 2张相同点数的牌 | 8♥ 8♥ |
| **拖拉机** | 2个或更多连续的对子 | 9♠9♠ 10♠10♠ |
| **甩牌** | 同花色多张不同点数的牌 | 3♦ 5♦ K♦ |

---

## 主牌判定

一张牌是主牌的条件（满足任一即可）:

1. 花色是主花色（叫牌成功的花色）
2. 点数是当前等级（如打2，所有2都是主牌）
3. 大小王永远是主牌

**主牌大小顺序**（以打2、主牌♠为例）:
```
3♠ < 4♠ < ... < A♠           ← 主花色非等级牌
< 2♥ < 2♣ < 2♦              ← 副花色等级牌
< 2♠ 2♠                      ← 主花色等级牌（对子）
< 小王                        ← 小王
< 大王                        ← 大王（最大）
```

## 代码文件说明

| 文件 | 说明 |
|------|------|
| `game_manager.gd` | 游戏主控制器，管理整个流程 |
| `game_rules.gd` | 游戏规则引擎，牌型识别和比较 |
| `player.gd` | 玩家类，手牌管理和AI逻辑 |
| `card.gd` | 卡牌类，卡牌渲染和交互 |
| `deck.gd` | 牌堆管理，创建和洗牌 |
| `ui_manager.gd` | UI管理器，界面更新 |
